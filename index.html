Linux 动态链接库相关整理

Posted on 2022-03-30 Edited on 2022-04-15 In System

PingCAP（aka 贵司）自研了套部署工具 TiUP 来取代早前使用的重型部署工具 Ansible。实际使用过程中，有多次出现在打 patch 滚动重启过程中，TiFlash 节点行为不符合预期。后来发现，TiUP 打 patch 过程中，没有停进程且是直接 tar 命令解压覆盖部署目录，原子性保障更是无从谈起。

v6.0 版本前 TiFlash 的部分模块包含动态加载 共享库（aka 动态库）的行为，这类行为会被搜索路径下的文件影响。如果存在某个时刻引入的二进制库不兼容，则直接影响程序的行为。

因为是用 tar 命令解压覆盖，反而是没有出现老生常谈的 cp 覆盖动态库引起 Segmentation fault 的问题。

在此整理下相关的问题：

  • cp / tar 命令是如何工作的？
  • cp 覆盖正在被使用的动态库为何会引发异常？
  • 调用动态库内函数为何会产生不符合直觉的行为？
  • 动态库性能开销有多大？

浅析数据块排序

Posted on 2019-12-18 Edited on 2022-04-01 In HistoryBlogs

目前主流的数据库引擎可按照实现方式分为以下几种：B+ Tree、LSM、Delta Main。在类 LSM 的存储方案中，RocksDB 和 Kudu 之流通过 append 的方式写日志并定期重新组织，从而将写数据的压力平衡到读取的过程中。由于相邻的数据可能落在 Memtable 以及不同的 SST 中，在涉及到范围扫描时，需要对结构化/半结构化数据进行排序输出。因此，对于数据块排序过程的优化，也会对读性能产生重要影响。
